    **Store shape:**
    ```typescript
    interface DeckCardsState {
      deckId: number | null;
      cards: DeckCard[];
      viewMode: 'grid' | 'list';
      loading: boolean;
      error: string | null;
      loadForDeck: (deckId: number) => Promise<void>;
      addCard: (deckId: number, card: Card) => Promise<AddResult>;
      removeCard: (deckCardId: number) => Promise<void>;
      setViewMode: (deckId: number, mode: 'grid' | 'list') => Promise<void>;
    }
    type AddResult = { ok: true; deckCardId: number } | { ok: false; reason: 'already-in-deck' | 'storage-error' };
    ```

    **loadForDeck(deckId):**
    - Reads `db.decks.get(deckId)` → extracts `viewMode ?? 'list'`.
    - Reads `db.deckCards.where('deckId').equals(deckId).toArray()`.
    - Sets `{ deckId, cards, viewMode, loading: false, error: null }`.

    **addCard(deckId, card):**
    1. Singleton check against in-memory `state.cards`: if `state.cards.find(c => c.scryfallId === card.id)` exists AND `!isBasicLand(card)`, return `{ ok: false, reason: 'already-in-deck' }` without touching Dexie or Scryfall.
    2. Resolve originalReleaseDate via `resolveOriginalReleaseDate(card)` (see below). Non-blocking — any throw is caught and `null` is used.
    3. Open ONE `db.transaction('rw', [db.deckCards, db.deckChanges, db.decks], async () => { ... })`:
       - Insert DeckCard with `{ deckId, scryfallId: card.id, cardName: card.name, quantity: 1, isCommander: false, addedAt: now, originalReleaseDate }` → capture `newId`.
       - Insert DeckChange `{ deckId, type: 'add', cardName: card.name, scryfallId: card.id, timestamp: now }`.
       - Update `db.decks.update(deckId, { updatedAt: now })`.
    4. On transaction success: `set(state => ({ cards: [...state.cards, { id: newId, ...row }] }))`. Return `{ ok: true, deckCardId: newId }`.
    5. On transaction error: `set({ error: 'Could not add card. Check your browser storage settings and try again.' })`, return `{ ok: false, reason: 'storage-error' }`.

    **resolveOriginalReleaseDate(card):** internal helper.
    1. Cross-deck dedupe: `const existing = await db.deckCards.filter(dc => dc.scryfallId === card.id && dc.originalReleaseDate != null).first(); if (existing?.originalReleaseDate) return existing.originalReleaseDate;`
    2. Try `searchCards(<locked-query-string-from-probe>, { unique: 'prints', order: 'released', dir: 'asc' })`.
       - If `03-ORACLEID-PROBE.md` locked `oracleid:` → `` `oracleid:${card.oracle_id}` ``.
       - If locked `oracle_id:` → `` `oracle_id:${card.oracle_id}` ``.
       - If locked `prints_search_uri-parsing` → parse `card.prints_search_uri` URL, extract `q` param, pass that to searchCards.
    3. `const released = result.data[0]?.released_at;` — `Date | null | undefined`.
    4. If `released instanceof Date && !isNaN(released.getTime())` → return `released.toISOString().slice(0, 10)`. Else `null`.
    5. Catch any throw → `console.warn('[deck-cards-store] originalReleaseDate lookup failed', err); return null;`.

    **removeCard(deckCardId):**
    1. Find row in state: `const row = state.cards.find(c => c.id === deckCardId); if (!row) return;`
    2. Single transaction (rw over deckCards + deckChanges + decks):
       - `db.deckCards.delete(deckCardId)`
       - `db.deckChanges.add({ deckId: row.deckId, type: 'remove', cardName: row.cardName, scryfallId: row.scryfallId, timestamp: now })`
       - `db.decks.update(row.deckId, { updatedAt: now })`
    3. `set(state => ({ cards: state.cards.filter(c => c.id !== deckCardId) }))`.

    **setViewMode(deckId, mode):**
    1. `await db.decks.update(deckId, { viewMode: mode, updatedAt: Date.now() });`
    2. `set({ viewMode: mode });`

## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Browser → Scryfall HTTPS | `searchCards(oracleid:...)` sends oracle_id to api.scryfall.com for prints lookup |
| Browser → IndexedDB | All deck writes travel through Dexie; same-origin isolated |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-03-03-01 | Denial of Service | Scryfall prints-lookup abuse via click-spam | mitigate | (1) scryfall-api library 100ms throttle [Phase 02.3 D-09]; (2) cross-deck dedupe short-circuits repeat oracle_ids; (3) UI disables (+) button during fetch (Plan 04); (4) unit test asserts exactly 1 call per addCard invocation. |
| T-03-03-02 | Tampering | Corrupt Date from scryfall-api reviver | mitigate | Explicit `released instanceof Date && !isNaN(released.getTime())` guard before formatting. Otherwise return null. |
| T-03-03-03 | Information Disclosure | oracle_id sent to Scryfall | accept | oracle_id is public catalog data; no PII. Same network pattern as Phase 02 card search. |
| T-03-03-04 | Denial of Service | Dexie transaction failure (quota exhaustion) | mitigate | addCard catches transaction error and returns `storage-error`. UI-SPEC error banner surfaces it. No deckChanges row is written if the transaction aborts (atomic guarantee from Dexie). |
| T-03-03-05 | Tampering | Duplicate deckCards rows via race | mitigate | In-memory singleton pre-check is racy across tabs but acceptable — IndexedDB is same-origin and users don't typically open two tabs of the same deck. Dexie transaction does not have a uniqueness index on (deckId, scryfallId), so a second add in a race WILL succeed. Accepted risk; add a UNIQUE index only if observed in practice. |
</threat_model>
