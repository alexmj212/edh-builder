# Phase 3 — UI Design Contract: Deck Building & Card Display

> Visual and interaction contract for frontend phases. Generated by gsd-ui-researcher, verified by gsd-ui-checker.

## Design System

| Property | Value |
|----------|-------|
| Tool | none |
| Preset | not applicable |
| Component library | none (hand-rolled Tailwind v4 components) |
| Icon library | inline SVG (no external icon library) |
| Font | system-ui (browser default, no custom font) |

No `components.json` found. Project uses Tailwind v4 with OKLCH custom tokens defined in `src/index.css`. Executor must reference the existing token set; no new library is introduced.

## Spacing Scale

Declared values (must be multiples of 4):

| Token | Tailwind | Value | Usage |
|-------|----------|-------|-------|
| xs | `gap-1` / `p-1` | 4px | Icon gaps, inline SVG padding |
| sm | `gap-2` / `p-2` | 8px | Compact element spacing, button inner padding, deck row gap, category header padding |
| md | `gap-4` / `p-4` | 16px | Default element spacing, card cell padding, error banners |
| xl | `gap-6` / `mt-6` | 24px | Section vertical rhythm |
| 2xl | `mt-8` | 32px | Major section breaks (search→deck pane gap) |
| 3xl | `py-16` | 64px | Empty/error state vertical centering |

Exceptions:
- Touch target minimum: `(+)` and `(×)` buttons must be `w-8 h-8` (32px) with a `p-1 -m-1` tap extension giving 44px effective target — matches existing `CardResultCell` pattern.
- Grid skeleton placeholder height is aspect-ratio-driven, not a fixed spacing token. Use `aspect-[488/680]` (`normal` size) for search results and `aspect-[146/204]` (equivalent ratio, `small` size 146×204) for deck grid thumbnails.

## Color

All values are OKLCH tokens from `src/index.css`. Light and dark variants are defined there; use token names only — never hardcode hex or rgb.

| Role | Token (light) | Token (dark) | CSS var | Usage |
|------|--------------|--------------|---------|-------|
| Dominant (60%) | oklch(0.97 0.01 250) | oklch(0.15 0.01 250) | `bg-background` | Page background, input backgrounds |
| Secondary (30%) | oklch(0.93 0.01 250) | oklch(0.20 0.01 250) | `bg-surface` | Deck column pane, category headers, card rows, `(×)` button background |
| Accent (10%) | oklch(0.50 0.15 250) | oklch(0.65 0.15 250) | `text-accent` / `bg-accent` | `(+)` button background, view toggle active state, focus rings |
| Destructive | oklch(0.50 0.20 25) | oklch(0.65 0.20 25) | `text-danger` / `border-danger` | `(×)` remove button on hover/focus (danger tint) |
| Surface hover | oklch(0.88 0.01 250) | oklch(0.25 0.01 250) | `bg-surface-hover` | Deck row hover, view toggle hover |
| Text primary | oklch(0.20 0.01 250) | oklch(0.93 0.01 250) | `text-text-primary` | All primary readable text |
| Text secondary | oklch(0.40 0.01 250) | oklch(0.70 0.01 250) | `text-text-secondary` | Metadata, counts, placeholders, disabled text |
| Border | oklch(0.80 0.01 250) | oklch(0.30 0.01 250) | `border-border` | All borders: pane dividers, card rows, category headers |

**Accent reserved for:** `(+)` add button (`bg-accent text-white`), view toggle active segment background, keyboard focus rings (`focus:ring-accent focus:ring-2`), hyperlink text (`text-accent hover:text-accent-hover`).

**Destructive tint on `(×)` remove button:** button is `text-text-secondary` at rest; `hover:text-danger focus:text-danger` on interaction. Background does not change at rest — only icon color shifts to danger on hover/focus. This makes the intent clear without being alarming in a full list of rows.

## Component Contracts

### Deck column header

```
<div class="flex items-center justify-between mb-4">
  <h2 class="text-xl font-semibold text-text-primary">
    Your Deck
    <span class="ml-2 text-sm font-normal text-text-secondary">Cards: {n}</span>
  </h2>
  <ViewToggle />  <!-- see below -->
</div>
```

Card count: `Cards: {n}` — always visible, no parentheses, no trailing denominator in Phase 3 (Phase 4 adds the `/100` goal). At zero: `Cards: 0`.

### View toggle (segmented control)

Rendered as two `<button>` elements side-by-side inside a `<div>` container. NOT a checkbox or radio — buttons with `aria-pressed`.

```
<div role="group" aria-label="Deck view" class="flex rounded-lg border border-border overflow-hidden text-sm">
  <button aria-pressed={viewMode === 'list'}
    class="px-2 py-1 font-semibold {active: bg-accent text-white | inactive: bg-surface text-text-secondary hover:bg-surface-hover}">
    List
  </button>
  <button aria-pressed={viewMode === 'grid'}
    class="px-2 py-1 font-semibold border-l border-border {active: bg-accent text-white | inactive: bg-surface text-text-secondary hover:bg-surface-hover}">
    Grid
  </button>
</div>
```

Active segment: `bg-accent text-white` (no border radius needed — overflow-hidden on parent clips corners).
Inactive segment: `bg-surface text-text-secondary hover:bg-surface-hover transition-colors`.
Focus ring: `focus:outline-none focus:ring-2 focus:ring-accent focus:ring-inset` on each button.

Persisted per-deck via `Deck.viewMode` (Dexie v4 additive field). Default: `'list'`.

### Scroll restoration on view switch

When the user switches between list and grid views, the deck column's own `scrollTop` is reset to `0`. No complex restoration needed — the position mapping between list rows and grid cells is not 1:1. Reset is the correct behaviour for v1.

### Type-category headers (list view only)

One `<section>` per non-empty category. Empty categories are omitted.

```
<section aria-label="{CategoryName} ({count})">
  <div class="flex items-center justify-between px-2 py-2 bg-surface border-b border-border sticky top-0 z-10">
    <span class="text-xs font-semibold text-text-secondary uppercase tracking-wide">
      {CategoryName}
    </span>
    <span class="text-xs text-text-secondary">{count}</span>
  </div>
  {/* card rows */}
</section>
```

Category order (fixed, not alphabetical):
1. Creatures
2. Planeswalkers
3. Instants
4. Sorceries
5. Artifacts
6. Enchantments
7. Lands

Category headers are sticky within the scrolling deck column (`sticky top-0`) so the user knows which section they are in when scrolling.

Count format: integer only, e.g. `4` not `4/99`. No category is shown when it has 0 members.

### Deck row (list view)

One `<li>` per card in the category.

```
<li class="flex items-center gap-2 px-2 py-2 hover:bg-surface-hover transition-colors group">
  <!-- small art thumbnail (optional but recommended) -->
  <img src="{small_image_url}" alt="" aria-hidden="true"
    class="w-8 h-8 rounded object-cover flex-shrink-0 bg-surface" loading="lazy" />
  <!-- card name -->
  <span class="flex-1 text-sm text-text-primary line-clamp-1">{card.name}</span>
  <!-- mana cost (secondary) -->
  <span class="text-xs text-text-secondary flex-shrink-0">{card.mana_cost}</span>
  <!-- remove button -->
  <button type="button"
    aria-label="Remove {card.name} from deck"
    class="w-8 h-8 flex items-center justify-center rounded
           text-text-secondary hover:text-danger focus:text-danger
           opacity-0 group-hover:opacity-100 focus:opacity-100
           focus:outline-none focus:ring-2 focus:ring-accent
           transition-all flex-shrink-0">
    <!-- × SVG 12×12 -->
  </button>
</li>
```

Notes:
- Remove button uses `opacity-0 group-hover:opacity-100 focus:opacity-100` to avoid visual noise. It IS keyboard-accessible at all times (focusable, just visually hidden until hover/focus).
- `aria-label="Remove {card.name} from deck"` — fully qualified, not just "Remove".
- Thumbnail in list row: `w-8 h-8` (32px square), `object-cover`, rounded, `loading="lazy"`. Use `small` image URI. If image fails to load, fallback to a `bg-surface rounded` colored block (CSS background — no broken-image icon).

### (×) Remove button — all states

| State | Classes |
|-------|---------|
| Default (list row not hovered) | `opacity-0` (visually hidden, still in tab order) |
| Row hovered | `opacity-100 text-text-secondary` |
| Button focused | `opacity-100 text-danger ring-2 ring-accent` |
| Button hovered | `opacity-100 text-danger` |
| Pressed (action fires) | No special state — row disappears immediately on confirm |

The `(×)` SVG icon: `viewBox="0 0 12 12"`, paths `M2 2 L10 10 M10 2 L2 10`, `stroke="currentColor"`, `strokeWidth="2"`, `strokeLinecap="round"`. Size: `w-3 h-3` (12px).

No confirmation dialog for remove in v1. Action is immediate. `deckChanges` log captures the event. Undo is deferred.

### (×) Remove button in grid view

In grid view, the remove button appears in the top-right corner of each card image, always visible (not hover-reveal), because the grid is smaller and hover states are harder to target on touch.

```
<button type="button"
  aria-label="Remove {card.name} from deck"
  class="absolute top-2 right-2 w-8 h-8 rounded-full
         bg-background/80 flex items-center justify-center
         text-text-secondary hover:text-danger focus:text-danger
         focus:outline-none focus:ring-2 focus:ring-accent">
  <!-- × SVG 12×12 -->
</button>
```

### (+) Add button — CardResultCell states

| State | Classes | Copy |
|-------|---------|------|
| Default (card not in deck) | `bg-accent opacity-90 cursor-pointer` | — (SVG `+`) |
| Hover | `bg-accent-hover opacity-100` | — |
| Focus | `ring-2 ring-accent ring-offset-1 ring-offset-background` | — |
| Loading (fetching originalReleaseDate) | `opacity-50 cursor-not-allowed` + spinner replaces `+` icon | — |
| Already in deck (non-basic duplicate) | `opacity-40 cursor-not-allowed` | tooltip "Already in deck" |
| Basic land (always addable) | Same as Default — no disabled state for basics | — |

Disabled state for duplicates: `disabled` attribute set, `aria-label="Already in deck"`, `title="Already in deck"`. The button is visually opaque-dimmed (`opacity-40`), not hidden. This matches the existing disabled pattern on the button (`CardResultCell.tsx:35`).

Loading state: replace the `+` SVG with the existing `<Spinner />` component from `CardSearchSection.tsx`. Duration is the time for the Scryfall prints lookup. Non-blocking — if lookup fails, button returns to Default state and card is still added.

### Grid view (deck column)

Cards shown as thumbnails using Scryfall `small` image (146×204px natural size).

```
<div class="grid grid-cols-3 gap-2 p-2">
  {cards.map(card => (
    <div class="relative rounded overflow-hidden bg-surface">
      <div class="aspect-[146/204]">
        <!-- skeleton shown before image loads -->
        <div class="absolute inset-0 bg-surface animate-pulse" />
        <img src="{small_uri}" alt="{card.name}"
          loading="lazy" decoding="async"
          class="w-full h-full object-cover relative z-10"
          onLoad={showImage} />
      </div>
      <!-- remove button (always visible in grid) -->
      <button ...>×</button>
    </div>
  ))}
</div>
```

Skeleton: `bg-surface animate-pulse` div occupying the full `aspect-[146/204]` container. The skeleton is replaced (faded out) when `img.onLoad` fires. Use `opacity-0 group-[img-loaded]:opacity-100 transition-opacity` pattern or a simple boolean `useState` per cell.

Aspect ratio for `small`: 146×204 reduces to the same ratio as 488/680 (exactly `73/102`). Use `aspect-[146/204]` for clarity.

Grid columns: `grid-cols-3` in the 40% deck column. This gives approximately 3 cards wide at typical desktop widths.

### Commander strip at top of deck column

Reuse the existing `CommanderPanel` component (already handles primary + partner slots, `art_crop` image). Render it as the first visual element in the deck column, above the list/grid toggle. No additional wrapper needed — `CommanderPanel` already has `rounded-lg bg-surface border border-border p-4`.

The `CommanderPanel` is always visible at the top of the deck column regardless of list/grid view mode.

## Empty & Error States

### Deck column — no cards added yet

```
<div class="flex flex-col items-center justify-center py-16 text-center">
  <p class="text-text-secondary text-sm">No cards yet.</p>
  <p class="text-text-secondary text-sm mt-1">Add from search results on the left.</p>
</div>
```

Heading: "No cards yet."
Body: "Add from search results on the left."
No icon. No CTA button (the action is in the left column).

### Deck column — no commander selected (gate)

If no commander is set, the deck column shows:

```
<div class="flex flex-col items-center justify-center py-16 text-center">
  <p class="text-text-secondary text-sm">Pick a commander first.</p>
  <p class="text-text-secondary text-sm mt-1">Card search will be enabled once a commander is selected.</p>
</div>
```

This matches the existing `CardSearchSection` gating pattern.

### List view — empty category

Empty categories are not rendered. No "No creatures" placeholder inside a category.

### originalReleaseDate fetch failure

Non-blocking — card is still added. UI consequence: none visible to user. A `console.warn` is emitted. No toast, no error banner.

### Scryfall error during add

If the Scryfall prints lookup fails AND the add itself succeeds (add is non-blocking on this lookup), no error is shown. If the card add to Dexie fails (storage error):

```
<div role="alert"
  class="border-l-4 border-danger bg-surface rounded-r-lg p-4 text-sm text-text-primary mt-2">
  Could not add card. Check your browser storage settings and try again.
</div>
```

Matches the existing error pattern from `CardSearchSection`.

## Keyboard Navigation

Tab order within the deck column (top to bottom):

1. View toggle — List button
2. View toggle — Grid button
3. (Commander panel — existing tab order unchanged)
4. Each deck row: card name (not interactive), then `(×)` remove button
5. In grid view: each card cell `(×)` button in reading order (left to right, top to bottom)

Focus rings: `focus:outline-none focus:ring-2 focus:ring-accent` on all interactive elements. Matches existing pattern from `CardSearchSection` inputs.

Tab order within search results (left column) — unchanged from Phase 2. The `(+)` add button on `CardResultCell` becomes reachable when enabled; it already has `tabIndex={0}` on the parent `div` — the button inside receives natural tab focus.

`Escape` key: no special handling in Phase 3 (no modals or drawers).
