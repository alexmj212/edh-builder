## Behavior Changes

**Zero.** This plan is a pure type + guard cleanup — every semantic branch of partner-detection, commander-store, and the four components resolves to the same runtime as before:

- `keywordsOf(card)` previously returned `[]` when keywords was absent; post-change, `card.keywords` is always defined (empty array per GameplayCard requirement). Runtime fixture coverage: test fakeCard defaults now seed `keywords: []` so this remains `[]` for minimal mocks.
- `typeLineOf(card)` previously returned `''` when type_line was absent; post-change, `card.type_line` is always a string (may be `''` if the fixture supplies it). Same runtime semantics.
- `oracleTextOf(card)` previously returned `''` when both oracle_text and card_faces were absent; post-change, returns `''` identically (oracle_text is `string | null | undefined` so the `typeof === 'string'` check is retained).
- `areCompatiblePartners` named case previously guarded `'name' in secondary`; post-change, `secondary.name` is always a string — a `''` secondary cannot match the partnerName regex match, so the branch returns false identically.
- Components that previously dug `(card as unknown as { name: string }).name` now just read `card.name` — same field, same value, zero runtime delta.
- 189/189 vitest green confirms no behavior regressions.

## Decisions Made

All plan mandates honored:

- **Card type as the single app-wide surface:** consumers import `type { Card }` from `src/lib/scryfall` — no ScryfallCard anywhere in the retype zone.
- **In-guards removed where fields are required:** partner-detection's `'keywords' in card` / `'type_line' in card` / `'name' in secondary` guards all dropped — RESEARCH §Card Type Parity Matrix confirms GameplayCard requires these fields.
- **Helper function removal:** commander-store's colorIdentityOf / idOf / nameOf trio deleted; direct field access replaces them at call sites.
- **Bridge cast removal:** `as never` in scryfall-queries.ts and commander-store.ts removed once partner-detection accepts Card directly.
- **CachedCard retype:** types/card.ts cardJson field is now `Card` — card-cache.ts no longer needs the `CachedCard['cardJson']` indexed-type workaround from plan 02.3-02.

**Scope boundary respected:**
- `src/lib/scryfall-client.ts` + `.test.ts` — UNCHANGED. They still reference `@scryfall/api-types` (plan 02.3-05 deletes them).
