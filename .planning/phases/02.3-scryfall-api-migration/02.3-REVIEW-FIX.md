## Fixed Issues

### WR-1: `validatedData` discards the Zod-parsed output; the cast is redundant if schema strengthens

**Files modified:** `src/lib/scryfall.ts`
**Commit:** `4ea3669`
**Applied fix:** Replaced `CardArraySchema.parse(raw); return raw as Card[]` with `const parsed = CardArraySchema.parse(raw); return parsed as unknown as Card[]`. The double-cast through `unknown` is required because Zod's `passthrough()` inference produces `objectOutputType<...>[]` which TypeScript does not deem sufficiently overlapping with the library's full `Card` type. This matches the review's intent: the code now consumes the parsed output, so future schema tightening (e.g., `.transform()` or removal of `.passthrough()`) will correctly affect the returned array. Tests continue to pass (14/14 in `scryfall.test.ts`).

### WR-2: `searchPartnersFor` 'none' case allocates a real `MagicPageResult` with empty query

**Files modified:** `src/lib/scryfall.ts`, `src/lib/scryfall.test.ts`
**Commit:** `1c93076`
**Applied fix:** Replaced the `Cards.search('', { order: 'edhrec' })` allocation in the `searchPartnersFor` 'none' branch with an inert object cast: `{ next: () => Promise.resolve<Card[]>([]), hasMore: false, count: 0 } as unknown as MagicPageResult<Card>`. The synthesized handle has a safe `next()` that resolves to `[]` without any network I/O, eliminating the latent risk that a future caller could accidentally trigger a bad-request HTTP call to `api.scryfall.com/cards/search?q=`. Added explanatory comment documenting the rationale. Strengthened the corresponding test to assert `Cards.search was NEVER invoked` (via `searchSpyRef.current`) in the 'none' case, plus a new assertion that `result._page.next()` resolves to `[]` with no I/O. All 14 tests still pass.

### WR-3: No test for the Zod validation boundary rejecting malformed API responses

**Files modified:** `src/lib/scryfall.test.ts`

Both tests pass. Test count increased from 14 → 16. These tests now pin the behavior so that any future schema evolution (e.g., removing `.passthrough()`) cannot silently discard valid cards instead of throwing.
