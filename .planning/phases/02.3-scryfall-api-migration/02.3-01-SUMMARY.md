### D-08 — AbortSignal path

**Status:** PASS — **D-04 option 3 (promise-race wrapper) is ratified.**

**Evidence:**

- Library surface inspection (`node_modules/scryfall-api/dist/scryfall-api.d.ts`) shows no `setFetch`, `configure`, `fetcher`, or `AbortSignal` hook — NO native abort support, confirming RESEARCH §AbortSignal Integration Path.
- Probe demo: invoked `Cards.byId(...)` then raced it against a pre-aborted `AbortController` via a local `abortable(p, signal)` helper. Result: `{ rejected: true, errorName: 'AbortError' }`. The Promise wrapper rejected synchronously-via-microtask as designed. Underlying fetch may still complete in the background; that matches Phase 2's existing `card-search-store` semantics (stale results discarded by module-level `AbortController`).

**Locked downstream shape for 02.3-02:** a single `abortable<T>(p: Promise<T>, signal?: AbortSignal): Promise<T>` helper in `src/lib/scryfall.ts`. Every wrapper export (`searchCards`, `fetchCardById`, `fetchNextPage`, `searchCommanders`, `searchPartnersFor`) funnels through it — no ad-hoc per-call `Promise.race`.

### D-09 — Rate limit (shared, 100ms)

**Status:** PASS — library ships a module-global hard-coded 100ms throttle.

**Evidence:** Probe fired two sequential `Cards.byId` calls back-to-back:

```
gap1_ms: 42   (first call — no prior request timestamp, so throttle is a no-op)
gap2_ms: 100  (second call — throttle waited the required 100ms)
expected_min_ms: 100
```

`gap2_ms` hitting exactly 100ms (not 101ms, not network-jitter noise) is consistent with RESEARCH §Rate-Limit Throttle Semantics: the library implements `await sleep(Math.max(0, rateLimit - (Date.now() - lastQuery)))` where `rateLimit = 100` and `lastQuery` is a module-closed-over variable. Because `lastQuery` lives in the module scope (not an instance field), this throttle is **shared across all callers** — exactly the D-01 requirement.

**Cleanup trail for 02.3-02:** delete `MIN_GAP_MS`, `lastRequestTime`, `waitForRateLimit`, and `__resetRateLimit` from the legacy `src/lib/scryfall-client.ts` — the library supersedes them.

### D-10 — Gap tolerance (query builders)

**Status:** ACCEPTED GAP (not a blocker).

Library ships **zero** query builders — confirmed by RESEARCH §Query Builder Coverage Matrix and by probe: all three of our current builders (`buildSearchQuery`, `buildCommanderSearchQuery`, `buildPartnerQuery`) have no library counterpart. This is explicitly ALLOWED per CONTEXT §D-10.

**Resolution:** All three builders move to a new local module `src/lib/scryfall-queries.ts` (D-03 fallback) in plan 02.3-02. Probe confirmed our existing `buildCommanderSearchQuery` output string works verbatim against the live API (2926 hits), and the named-partner query (`!"Tymna the Weaver"`) returned exactly one card (`hitCount: 1, firstName: 'Tymna the Weaver'`), validating D-10's partner-variant handling.

## Locked Decisions for Downstream Plans

1. **Wrapper location:** `src/lib/scryfall.ts` (fresh file). Delete `src/lib/scryfall-client.ts` in plan 02.3-05.
2. **Abort path:** promise-race wrapper, single `abortable<T>()` helper in `src/lib/scryfall.ts`, every call site funnels through `searchCards` / `fetchCardById` / `fetchNextPage` / `searchCommanders` / `searchPartnersFor`.
3. **404 handling:** wrapper re-throws `new Error('Card not found: ' + id)` when `Cards.byId` returns `undefined`. Preserves existing call-site contract in `commander-store.loadForDeck`.
4. **Pagination:** new `SearchResult { data, hasMore, totalCards, _page: MagicPageResult<Card> }` shape. `_page` is opaque handle carried from `searchCards` → `fetchNextPage`. Replaces the `next_page: string` URL in the current `ScryfallList`.
5. **Query builders:** ALL THREE kept locally in new `src/lib/scryfall-queries.ts`. Library covers none.
6. **Zod envelope:** KEEP `ScryfallListSchema` at wrapper boundary per D-05. Runtime safety net still needed (memory: `feedback_verify_external_apis`).
7. **Throttle cleanup:** Delete `MIN_GAP_MS`, `lastRequestTime`, `waitForRateLimit`, `__resetRateLimit` — library supersedes.
8. **Card type:** `import type { Card } from 'scryfall-api'`; re-export from `src/lib/scryfall.ts` as the single type import point.
9. **Fixture typing:** `type FixtureCard = Pick<Card, ...fields the app reads>` in `e2e/helpers/stubScryfall.ts`. One `asCard()` helper wraps the single `as unknown as Card` cast; three call sites route through it.
10. **CI typecheck:** Add `npm run typecheck` step to `.github/workflows/e2e.yml` before the Playwright step (plan 02.3-04).
11. **Build script restoration timing:** `tsc -b && vite build` is restored in plan 02.3-05 AFTER `src/lib/scryfall-client.ts` deletion — restoring earlier would fail `tsc -b` because tsconfig.app.json includes all of `src/` with no exclusion.

## Known Gaps (Accepted — NOT blockers per D-10)

- **Library ships NO query builders** → kept in `src/lib/scryfall-queries.ts` (D-03 fallback).
- **Library has NO native AbortSignal** → promise-race wrapper (D-04 option 3). Underlying fetches complete in background; observable behavior matches SRCH-05.
- **Library rate-limit sleep is NOT abort-cancellable** → accepted per CONTEXT D-04. Tracked as Risk R4 in RESEARCH.
- **Library auto-parses `released_at` to `Date`** → NOT a 02.3 concern, flag for Phase 3 BUILD-08 planner.
- **Library's 3-retry-on-5xx behavior** → better than our current fail-immediately; accepted.
- **Library has NO User-Agent configuration hook** → probe identifies via bare `fetch()` ping; app-runtime requests go through library without identifying UA (acceptable for browser SPA — browser sends its own UA). Node-side tooling (future CI scripts) should prefer direct `fetch()` if UA identification is required.
