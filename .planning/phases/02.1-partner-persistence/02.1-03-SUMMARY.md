## Decisions Made

- **Single test-only commit.** The plan is nominally TDD, but Plan 02 already shipped the component call-site changes (`void clearPartner(deckId)` / `void setPartner(deckId, card)`) and the store now persists. A RED phase would have required reverting those changes, which the plan explicitly told us NOT to do. We wrote the tests, they immediately passed against the already-correct implementation, and we committed them as `test(02.1-03): ...` — pure regression coverage, no production code touched.
- **Await two `Promise.resolve()` turns after the Remove-partner click.** `clearPartner` awaits `db.decks.update` (one microtask) and then calls `set()` (another). Two `Promise.resolve()` awaits settle both without needing RTL's `waitFor`, which fights with `vi.useFakeTimers` elsewhere in the repo and is overkill here.
- **Pre-`render` call to `loadForDeck`.** The plan explicitly called out that a remount dance is not needed — calling `loadForDeck` before `render` mirrors how `DeckWorkspace` actually sequences mount-time hydration in production.
- **Re-used existing `fakeCard` helper verbatim.** Seeding `keywords: ['Partner']` makes `detectPartnerType` return `'generic'`, which activates the partner slot (not disabled) so the partner FullCard branch renders.
