# Phase 03.1 Plan 02: DeckColumn Scroll-Ref Lift Summary

Close UI-REVIEW Fix #2 by lifting the view-toggle scroll-reset target from DeckColumn's (non-scrolling) inner div to DeckWorkspace's real `flex-[2]` scroll container, and drop the redundant inner `overflow-y-auto` (UI-REVIEW Pillar 5 double-scroll advisory) in the same atomic change.

## What Changed

**`src/components/DeckColumn.tsx`**
- Added optional `onViewToggle?: () => void` to `DeckColumnProps`.
- Destructured `onViewToggle` from props.
- Removed `scrollRef` declaration (the misnamed/mistargeted ref).
- `handleToggle` now calls `onViewToggle?.()` after `setViewMode(deckId, mode)` instead of writing to its own scrollTop.
- Root `<div>` no longer carries `ref={scrollRef}` or `overflow-y-auto` — single scroll context delegated upward.

**`src/components/DeckWorkspace.tsx`**
- Imported `useCallback`, `useRef` alongside `useEffect`.
- Declared `outerScrollRef = useRef<HTMLDivElement>(null)`.
- Declared `handleViewToggle = useCallback(() => { outerScrollRef.current && (scrollTop = 0) }, [])`.
- Attached `ref={outerScrollRef}` to the `flex-[2]` wrapper (the real scroll container).
- Passed `onViewToggle={handleViewToggle}` to `<DeckColumn />`.

**`src/components/DeckColumn.test.tsx`**
- `renderDeckColumn` helper extended to accept optional `onViewToggle` param.
- Removed obsolete `resets scrollTop to 0 on view toggle` test (pinned to the removed inner overflow container).
- Added `describe('view-toggle scroll reset (Phase 03.1 UI polish)')` with two tests:
  1. `calls onViewToggle when user switches to Grid view` — seeds list-mode viewport with a card, clicks Grid, asserts `onViewToggle` called exactly once AND `setViewMode` called with `(1, 'grid')`.
  2. `calls onViewToggle when user switches to List view` — grid-mode setup, clicks List, symmetric assertions.
