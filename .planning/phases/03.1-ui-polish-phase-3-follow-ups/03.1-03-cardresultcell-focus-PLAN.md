<objective>
Fix UI-REVIEW finding #3: `CardResultCell`'s outer `div` has `tabIndex={0}` but renders no visible focus state. The decision (per .continue-here.md and the planner's reading of the outer-div role) is to ADD a focus ring to the outer container rather than remove the tab stop — the tab stop provides a useful "focus the whole card" affordance that pairs with the existing `focus-within:opacity-100` overlay reveal and with screen-reader navigation. The overlay already fades in on focus-within; now the card boundary itself gets a visible ring when keyboard focus lands on the outer div directly.

Purpose: Close WCAG 2.1 2.4.7 failure flagged in 03-UI-REVIEW.md Pillar 2 (Visuals) for tabbable outer container.

Output:
- `src/components/CardResultCell.tsx` outer `div` carries `focus:outline-none focus:ring-2 focus:ring-accent`
- `src/components/CardResultCell.test.tsx` gains a regression assertion for the focus-ring classes on the outer container

Decision recorded: KEEP `tabIndex={0}` on the outer div; ADD focus ring. Rationale: the outer tab stop + `focus-within` overlay pattern is intentional (keyboard users can preview the card without moving focus into the `(+)` button); removing the tab stop would regress that affordance.
</objective>
