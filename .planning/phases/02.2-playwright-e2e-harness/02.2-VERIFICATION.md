None of the above patterns block the phase goal. All are documented warnings for future cleanup.

**Severity assessment of WR-01 (build script):** The `tsc -b` removal is a pre-existing issue with `@scryfall/api-types@1.0.0-alpha.4` that predates Phase 02.2 (documented in Phase 02 02-01-SUMMARY.md). The executor's decision to surface it by separating `vite build` from `tsc -b` — and adding an explicit `typecheck` script — is the correct approach given the upstream alpha package constraint. This does not regress Phase 02.2's goal (the harness runs locally and in CI, and `npm run typecheck:e2e` covers the e2e layer in CI if added). The gap is the absence of a `typecheck:e2e` step in `e2e.yml`, which means e2e helper type errors also miss CI coverage. This is a separate concern from the alpha package issue and warrants a follow-up.
