<threat_model>
- **T-17 (high): GitHub Actions workflow uses `pull_request_target`** — that
  trigger runs with write permissions on the base repo's secrets AND checks
  out the PR head SHA, which would let any fork PR exfiltrate secrets.
  Mitigation: workflow uses `pull_request` (NOT `pull_request_target`).
  Acceptance criteria greps for this. **Block on finding `pull_request_target`.**
- **T-18 (high): Workflow grants write permissions by default** — GitHub's
  default token scope is write. Mitigation: set `permissions: contents: read`
  at the workflow top level. Acceptance criteria enforces it.
- **T-19 (medium): Artifact upload includes `.env*` or secrets** — Mitigation:
  upload path is explicitly `playwright-report/` only (NOT `./` or workspace-wide).
  Traces/videos do not embed process env.
- **T-20 (low): Action versions pinned to mutable tags** — `@v5` is a major-tag
  ref that GitHub maintains; acceptable risk. For maximum safety one would
  pin to SHA; not required for v1 per RESEARCH §Standard Stack.
- **T-21 (medium): `npm ci` runs post-install scripts for transitive deps** —
  low marginal risk given the locked package.json; Playwright is from Microsoft.
  Mitigation: trust the lockfile + dependabot updates + let Plan 01's
  dependency-confusion mitigation (verified repo) stand.
- **T-22 (low): CI runs on forks without fork-trust gating** — `pull_request`
  event on public repos triggers for forks but does NOT expose secrets (the
  forked workflow runs with a read-only token and no access to repo secrets).
  Safe default.

Two `high` findings (T-17, T-18) are both blocked by the canonical skeleton
and verified in acceptance criteria — proceed.
</threat_model>

    ```yaml
    # .github/workflows/e2e.yml
    # Source: https://playwright.dev/docs/ci-intro (adapted with least-privilege permissions)
    name: E2E (Playwright)

    on:
      push:
        branches: [main]
      pull_request:
        branches: [main]

    # Least privilege — this workflow only needs to read the repo and upload artifacts.
    permissions:
      contents: read

    # Cancel stale runs for the same ref to save CI minutes.
    concurrency:
      group: e2e-${{ github.workflow }}-${{ github.ref }}
      cancel-in-progress: true

    jobs:
      e2e:
        name: Playwright (chromium)
        timeout-minutes: 30
        runs-on: ubuntu-latest
        steps:
          - name: Checkout repository
            uses: actions/checkout@v5

          - name: Setup Node.js
            uses: actions/setup-node@v5
            with:
              node-version: lts/*
              cache: 'npm'

          - name: Install npm dependencies
            run: npm ci

          - name: Install Playwright browsers (chromium only)
            run: npx playwright install --with-deps chromium

          - name: Run Playwright suite
            run: npm run e2e
            env:
              CI: 'true'

          - name: Upload Playwright report on failure
            if: failure()
            uses: actions/upload-artifact@v4
            with:
              name: playwright-report
              path: playwright-report/
              retention-days: 14
    ```

    Do NOT use `pull_request_target`. Do NOT grant `permissions: write-all`.
    Do NOT upload artifacts unconditionally (wastes storage + might leak
    internal state on success).
