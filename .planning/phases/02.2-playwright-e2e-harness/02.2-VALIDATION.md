# Phase 02.2 — Validation Strategy

> Per-phase validation contract for feedback sampling during execution.

---

## Test Infrastructure

| Property | Value |
|----------|-------|
| **Framework** | `@playwright/test` (1.59.x) + existing `vitest` unit runner |
| **Config file** | `playwright.config.ts` (new — Wave 0 creates) |
| **Quick run command** | `npm run e2e -- <spec>` (single spec, headless, chromium) |
| **Full suite command** | `npm run e2e` (all specs, headless, chromium) |
| **Estimated runtime** | ~45 seconds full suite (cold-start smoke + 6 backfill specs) |

Existing unit/integration invariants remain verified by `npm test` (vitest). This VALIDATION doc focuses on the new E2E layer.

---

## Sampling Rate

- **After every task commit:** Run `npm run e2e -- <spec touched>` (or `npm test` if only unit-adjacent).
- **After every plan wave:** Run `npm run e2e` (full E2E suite) + `npm test` (vitest).
- **Before `/gsd-verify-work`:** Full E2E suite must be green AND `npm test` green AND `npm run build` clean.
- **Max feedback latency:** 60 seconds (single spec), 120 seconds (full suite).
