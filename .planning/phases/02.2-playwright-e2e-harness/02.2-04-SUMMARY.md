## Decisions Made

- Clicked h3 heading directly in createDeck rather than the outer wrapper div; the original `locator('div').filter(...).first()` was selecting the grid wrapper div (no onClick) ahead of the DeckCardItem div (has onClick). Clicking the h3 correctly bubbles up to DeckCardItem's onClick handler.
- IDB assertions use raw `indexedDB.open()` inside `page.evaluate()` — no need to import Dexie into test context; DB name "EDHBuilder" and store "decks" confirmed from src/lib/db.ts before writing.
- Null checks use `toBeNull()` not `toBeFalsy()` — Phase 02.1 decision establishes that cleared partner fields are explicitly `null`, not `undefined` or falsy.

## Deviations from Plan

### Auto-fixed Issues

**1. [Rule 1 - Bug] Fixed createDeck helper to click h3 heading instead of ambiguous div locator**
- **Found during:** Task 1 (first test run — both specs failed with URL mismatch at createDeck step)
- **Issue:** `page.locator('div').filter({ has: heading }).first()` selected the grid wrapper div (which contains the DeckCardItem h3 transitively) instead of the DeckCardItem div. The grid div has no onClick handler, so clicking it did not trigger `navigate('/decks/:id')`. The URL stayed at `/`.
- **Fix:** Changed to `page.getByRole('heading', { level: 3, name }).click()` — clicking the h3 directly; browser bubbles the click event to the DeckCardItem div's `onClick={onSelect}` handler which calls `navigate`.
- **Files modified:** `e2e/helpers/commanderFlows.ts`
- **Verification:** All 7 specs pass after fix; `toHaveURL(/\/decks\/\d+$/)` assertion in createDeck succeeds
- **Committed in:** `5bfa663` (Task 1 commit — bundled with specs 01+02)

## Issues Encountered

- The `createDeck` helper from Plan 02 had a latent bug: `locator('div').filter({ has: heading }).first()` is ambiguous when multiple ancestor divs all contain the heading. The `.first()` selected the outermost (grid) div, not the DeckCardItem div. This was only discoverable when running actual specs that call `createDeck` and then assert on URL navigation.
