|-------|-------|-------|

## Component Inventory

### 1. WorkspaceLayout (new)

Route: `/decks/:id`. Renders inside existing `<Layout>`. Single vertical scroll — no tabs.

Structure (top to bottom):
```
WorkspaceHeader        — back link + deck name
CommanderPanel         — commander slot(s) + search
CardSearchSection      — filters + locked chip + results grid
DeckPlaceholder        — "Deck building coming in Phase 3" stub
```

### 2. WorkspaceHeader

- Left: `←` + "Back to decks" link — `text-accent hover:text-accent-hover text-sm font-medium`
- Center/Right: Deck name in `text-2xl font-bold text-text-primary`
- Bottom border: `border-b border-border` to visually separate from content
- Height: 56px (`py-4` with header text)
- Back link uses `<Link to="/">` from react-router-dom

### 3. CommanderPanel

Container: `rounded-lg bg-surface border border-border p-4` (matches DeckList card style)

**Empty state (no commander selected):**
- Art area: `rounded bg-surface-hover` block, 160px (desktop) / 120px (mobile) height, full width
  - Centered copy inside: "No commander selected" in `text-text-secondary text-sm`
- Below art: "Pick a Commander" search input + results list (see item 4)

**Selected state (single commander):**
- Art area: `<img>` using `image_uris.art_crop` — `object-cover w-full h-40 sm:h-48 rounded-t-lg`
- Below art: commander name in `text-label font-semibold`, mana cost in monospace letters, type line in `text-text-secondary text-sm`
- "Change" button: `text-sm text-text-secondary hover:text-text-primary` — replaces the search input

**Partner layout (two-slot, side-by-side, D-03):**
- `grid grid-cols-2 gap-4`
- Left slot: primary commander (always active)
- Right slot: "Partner (optional)" — disabled until primary is partner-eligible
  - Disabled styling: `opacity-50 cursor-not-allowed border-dashed border-border bg-surface`
  - Disabled label: "Partner (optional)" in `text-text-secondary text-sm text-center`
  - When partner-eligible: slot activates — remove opacity-50, border becomes solid, search input appears
  - Each slot is a self-contained mini-panel with art + name below

### 4. CommanderSearch

Shown below art area when in empty or "Change" state.

- Input: full-width `bg-background border border-border rounded-lg px-3 py-2 focus:outline-none focus:ring-2 focus:ring-accent text-text-primary placeholder-text-secondary`
- Placeholder: "Search for a commander..."
- Results: vertically stacked list below input (not a grid), each row = art_crop thumbnail (48×48 rounded) + name + type line
  - Row: `flex items-center gap-3 py-2 px-3 rounded hover:bg-surface-hover cursor-pointer`
  - Thumbnail: `w-12 h-12 rounded object-cover bg-surface-hover` (shimmer when loading)
  - Name: `text-sm font-semibold text-text-primary`
  - Type: `text-xs text-text-secondary truncate`
- Loading state: 4 skeleton rows, each `h-14 rounded bg-surface-hover animate-pulse`
- Empty input: show EDHREC default list (same row style as results)
- Zero results: inline "No commanders match your search." in `text-text-secondary text-sm py-4 text-center`

### 5. ColorIdentityChip (D-07)

Always visible above card search results. Never hidden.

- Container: `flex items-center gap-2 mb-4`
- Copy: "Color identity:" in `text-sm text-text-secondary`
- Identity display: one colored square per pip — `w-5 h-5 rounded-sm text-xs font-bold flex items-center justify-center`
  - W = bg white, text-gray-800: "W"
  - U = bg-blue-600, text-white: "U"
  - B = bg-gray-800 border border-gray-600, text-white: "B"
  - R = bg-red-600, text-white: "R"
  - G = bg-green-600, text-white: "G"
  - Colorless (empty identity): single "C" square in bg-gray-500 text-white
- When no commander: entire chip reads "Filtered to: Pick a commander first" in `text-text-secondary text-sm italic`
  - No color squares shown
- Tooltip on click: inline expand below chip — `text-xs text-text-secondary bg-surface border border-border rounded p-2 mt-1`
  - Copy: "Results are limited to cards legal in Commander and within your commander's color identity."
  - Dismiss on click away or second click

**No external mana symbol library.** Colored letter squares are self-contained.

### 6. CardSearchSection

Container: `mt-8 space-y-4`

Section heading: "Card Search" — `text-xl font-semibold text-text-primary mb-4` (matches WorkspaceHeader section pattern)

**Filter row (D-06):**
- `grid grid-cols-1 gap-3 sm:grid-cols-2 lg:grid-cols-4`
- Name input: placeholder "Card name..."
- Type input: placeholder "Card type..."
- Text input: placeholder "Oracle text..."
- Color filter: `flex gap-2 items-center` — five toggle buttons (W/U/B/R/G) matching ColorIdentityChip pip styles, toggled on = ring-2 ring-accent

**Disabled state (no commander, D-07):**
- All filter inputs: `opacity-50 pointer-events-none`
- Above filters: inline banner — `text-sm text-text-secondary bg-surface border border-border rounded-lg p-3 mb-3`
  - Copy: "Pick a commander first to start searching cards."

### 7. CardResultsGrid (D-08)

- Grid: `grid grid-cols-2 gap-3 sm:grid-cols-3 md:grid-cols-4 lg:grid-cols-5 xl:grid-cols-6`
- Each cell: `relative rounded-lg overflow-hidden bg-surface border border-border cursor-pointer`

**Cell anatomy:**
- Image: `<img loading="lazy">` using `image_uris.normal` (488×680) — `w-full aspect-[488/680] object-cover`
- Skeleton: shown while image loads — `w-full aspect-[488/680] bg-surface-hover animate-pulse rounded`
- Hover/tap overlay (D-08): `absolute inset-0 bg-background/90 flex flex-col justify-end p-2 opacity-0 hover:opacity-100 transition-opacity duration-150`
  - Card name: `text-sm font-semibold text-text-primary line-clamp-1`
  - Mana cost: `text-xs text-text-secondary`
  - Type line: `text-xs text-text-secondary line-clamp-1 mt-0.5`
  - Oracle snippet: `text-xs text-text-secondary line-clamp-2 mt-1`
  - "Add" icon button (stub): `absolute top-2 right-2 w-8 h-8 rounded-full bg-accent flex items-center justify-center` — inline SVG "+" icon
    - Phase 2 behavior: `disabled` with `title="Add to deck — coming in Phase 3"`, `opacity-50 cursor-not-allowed`
    - Min touch target satisfied at 32px (w-8 h-8) — below 44px. Exception: add `p-1 -m-1` to expand hit area to 40px. This is acceptable given the overlay context.

**Touch/mobile (no hover):**
- First tap reveals overlay (toggle). Second tap does nothing (Add is stub-disabled).
- Overlay toggle: managed via `useState` on the cell.

### 8. Inline State Messages (D-15)

**Pattern:** All states render inline within their section — no toasts, no error boundaries for search errors.

| State | Location | Visual |
|-------|----------|--------|
| Empty input (commander search) | Below commander search input | `text-text-secondary text-sm text-center py-4`: "Start typing to search for a commander." |
| Empty input (card search) | Above results grid area | No message — show nothing (search hasn't fired) |
| Loading | Above results (spinner) | `flex items-center gap-2 text-text-secondary text-sm py-4`: spinning SVG + "Searching..." |
| Zero results | Where grid would appear | `text-text-secondary text-sm text-center py-8`: "No cards match your filters." |
| API error | Above results | `border-l-4 border-danger bg-surface rounded-r-lg p-3 flex items-start gap-3`: error icon + message text + "Retry" `text-accent` button |
| Network offline | Above results | Same banner pattern: "You appear to be offline. Check your connection and try again." + "Retry" button |
| Deck not found | Full workspace area | `text-text-secondary text-center py-16`: "Deck not found." + "← Back to decks" `text-accent` link |

**Loading spinner:** 20px inline SVG circle with `animate-spin`, `text-text-secondary`. No third-party spinner library.

### 9. LoadMoreButton (D-10, SRCH-07)

Pagination: load-more button (chosen over intersection observer — simpler to test, explicit user intent, sufficient for 175-card pages).

- Position: centered below results grid, `mt-6 flex justify-center`
- Button: `px-6 py-2 rounded-lg bg-surface border border-border text-text-primary hover:bg-surface-hover text-sm font-medium transition-colors`
- Copy: "Load more results"
- Loading state: spinner replaces button text, button disabled
- Hidden when `has_more === false`

### 10. DeckPlaceholder

Stub section below card search for Phase 3.

- `mt-8 rounded-lg bg-surface border border-border border-dashed p-8 text-center`
- Copy: "Deck cards will appear here." in `text-text-secondary text-sm`
- Not interactive — no click handler.

## Interaction Contracts

### Commander Selection Flow

1. User arrives at `/decks/:id` — WorkspaceHeader renders, CommanderPanel shows empty state
2. User types in commander search input → debounce 400ms → fire Scryfall query → results list appears
3. User clicks a commander row → commander set → art_crop banner replaces empty area → search input disappears
4. If commander has `Partner`/`Friends Forever`/`Choose a Background` in keywords/oracle_text:
   - Partner slot activates (opacity-50 removed, becomes interactive)
   - Partner slot search input appears pre-filtered to valid pairings only
5. User can click "Change" to return to search input (clears partner slot too if one was set)

### Card Search Flow

1. No commander → all filter inputs are `pointer-events-none opacity-50`, banner shows "Pick a commander first"
2. Commander selected → inputs activate, ColorIdentityChip shows commander's pips
3. User types in any filter → 400ms debounce → compose `q` string → fire search → show spinner → show grid
4. User taps/hovers card → overlay slides in (opacity transition 150ms)
5. User clicks "Add" stub → no-op (disabled with tooltip in Phase 2)
6. User clicks "Load more" → fetch `next_page` URL → append results to grid (no page replacement)

### Routing

- `/` — DeckList (unchanged from Phase 1)
- `/decks/:id` — WorkspaceLayout
- Not found deck: show "Deck not found." inline with link back to `/`
- Browser back from workspace → DeckList (BrowserRouter handles this automatically)

## Accessibility Notes

- All interactive elements reachable by keyboard (tab order follows DOM order)
- Card overlay reveal also triggers on `:focus-visible` (for keyboard navigation)
- Commander search input: `aria-label="Search for a commander"`
- Card search inputs: `aria-label` per field (e.g., `aria-label="Filter by card name"`)
- ColorIdentityChip color squares: `aria-label="White"` (etc.) on each square
- Disabled inputs: `aria-disabled="true"` when no commander selected
- Error banners: `role="alert"` so screen readers announce them immediately
- Skeleton placeholders: `aria-busy="true"` on grid container while loading
